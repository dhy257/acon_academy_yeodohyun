# 03. 제네릭 기초와 와일드카드

## 학습 목표

- 간단한 제네릭 클래스를 작성할 수 있다.
- 제한된 제네릭 클래스를 작성할 수 있다.
- 간단한 제네릭 매서드를 작성할 수 있다.
- 매서드 인자에서 와일드카드를 사용하는 법을 이해할 수 있다.

## 1. 제네릭이 필요한 이유

제네릭은 타입을 나중에 정할 수 있게 해 주는 문법이다.

제네릭이 없으면 여러 타입을 담기 위해 `Object`를 많이 사용한다.

```java
class BoxObject {
    private Object item;

    public void setItem(Object item) {
        this.item = item;
    }

    public Object getItem() {
        return item;
    }
}
```

이 방식은 어떤 타입이든 담을 수 있지만 꺼낼 때 형변환이 필요하다.

```java
BoxObject box = new BoxObject();
box.setItem("hello");

String value = (String) box.getItem();
```

형변환을 잘못하면 실행 중 오류가 발생할 수 있다. 제네릭은 이런 문제를 컴파일 단계에서 줄여 준다.

## 2. 간단한 제네릭 클래스

```java
class Box<T> {
    private T item;

    public void setItem(T item) {
        this.item = item;
    }

    public T getItem() {
        return item;
    }
}
```

`T`는 타입을 담는 변수처럼 생각하면 된다. 실제 타입은 객체를 만들 때 정한다.

```java
Box<String> stringBox = new Box<>();
stringBox.setItem("apple");
String value = stringBox.getItem();

Box<Integer> intBox = new Box<>();
intBox.setItem(10);
Integer number = intBox.getItem();
```

`Box<String>`은 문자열만 담는 상자가 되고, `Box<Integer>`는 정수만 담는 상자가 된다.

## 3. 제네릭 클래스의 장점

- 같은 클래스를 여러 타입에 재사용할 수 있다.
- 형변환 코드가 줄어든다.
- 잘못된 타입을 넣으면 컴파일 단계에서 확인할 수 있다.

```java
Box<String> box = new Box<>();
box.setItem("hello");

// box.setItem(10); // 컴파일 에러
```

## 4. 제한된 제네릭 클래스

제네릭 타입에 아무 타입이나 들어오지 못하게 제한할 수 있다.

```java
class NumberBox<T extends Number> {
    private T value;

    public void setValue(T value) {
        this.value = value;
    }

    public T getValue() {
        return value;
    }
}
```

`T extends Number`는 `T` 자리에 `Number` 또는 `Number`의 자식 클래스만 올 수 있다는 뜻이다.

가능한 예시는 다음과 같다.

```java
NumberBox<Integer> intBox = new NumberBox<>();
NumberBox<Double> doubleBox = new NumberBox<>();
```

불가능한 예시는 다음과 같다.

```java
// NumberBox<String> stringBox = new NumberBox<>(); // 컴파일 에러
```

수업의 3D 프린터 예제로 보면 다음과 같다.

```java
class Material {
}

class Powder extends Material {
}

class Plastic extends Material {
}

class ThreeDPrinter<T extends Material> {
    private T material;

    public void setMaterial(T material) {
        this.material = material;
    }

    public T getMaterial() {
        return material;
    }
}
```

이 클래스는 `Material` 계열만 재료로 받을 수 있다.

## 5. 제네릭 매서드

제네릭은 클래스뿐 아니라 매서드에도 붙일 수 있다.

```java
class MyGenericMethod {
    public static <T> void printItem(T item) {
        System.out.println(item);
    }
}
```

매서드 앞의 `<T>`는 "이 매서드 안에서 T라는 타입 이름을 사용하겠다"라는 선언이다.

```java
MyGenericMethod.printItem("hello");
MyGenericMethod.printItem(100);
MyGenericMethod.printItem(true);
```

타입을 직접 적을 수도 있다.

```java
MyGenericMethod.<String>printItem("hello");
```

대부분은 컴파일러가 타입을 추론해 주므로 생략한다.

## 6. 값을 비교하는 제네릭 매서드

수업 예제처럼 배열 안에서 특정 값과 같은 값의 개수를 세는 매서드를 만들 수 있다.

```java
class CountUtil {
    public static <T> int count(T[] array, T target) {
        int count = 0;

        for (T item : array) {
            if (item.equals(target)) {
                count++;
            }
        }

        return count;
    }
}
```

사용 예시는 다음과 같다.

```java
Integer[] numbers = {1, 2, 1, 3, 1};
int result1 = CountUtil.count(numbers, 1);

String[] words = {"apple", "banana", "apple"};
int result2 = CountUtil.count(words, "apple");
```

하나의 매서드가 `Integer`, `String` 등 여러 타입에 재사용된다.

## 7. T와 ?의 차이

제네릭을 배울 때 `T`와 `?`가 헷갈릴 수 있다.

`T`는 매서드나 클래스 안에서 타입 이름을 정해서 계속 사용할 때 쓴다.

```java
public static <T> T getFirst(List<T> list) {
    return list.get(0);
}
```

여기서는 매개변수 타입과 리턴 타입이 같은 `T`로 연결된다.

`?`는 정확한 타입 이름은 모르지만 어떤 범위 안의 타입을 허용하고 싶을 때 쓴다.

```java
public static void printList(List<?> list) {
    for (Object item : list) {
        System.out.println(item);
    }
}
```

`List<?>`는 `List<String>`, `List<Integer>`, `List<Acorn>` 등을 받을 수 있다.

## 8. 와일드카드 기본형: List&lt;?&gt;

```java
public static void printList(List<?> list) {
    for (Object item : list) {
        System.out.println(item);
    }
}
```

`List<?>`는 타입을 정확히 모르는 리스트이다. 그래서 읽을 때는 `Object`로 꺼낼 수 있다.

```java
List<String> names = List.of("kim", "lee");
List<Integer> numbers = List.of(1, 2, 3);

printList(names);
printList(numbers);
```

## 9. extends 와일드카드

```java
public static void printNumbers(List<? extends Number> list) {
    for (Number number : list) {
        System.out.println(number);
    }
}
```

`? extends Number`는 `Number` 또는 `Number`의 자식 타입 리스트를 받겠다는 뜻이다.

```java
List<Integer> ints = List.of(1, 2, 3);
List<Double> doubles = List.of(1.5, 2.5);

printNumbers(ints);
printNumbers(doubles);
```

`extends`는 읽기 중심이라고 기억하면 좋다. 최소한 `Number`로는 꺼낼 수 있기 때문이다.

```java
Number number = list.get(0);
```

하지만 어떤 자식 타입 리스트인지 정확히 모르기 때문에 값을 추가하는 것은 제한된다.

## 10. super 와일드카드

```java
public static void addIntegers(List<? super Integer> list) {
    list.add(10);
    list.add(20);
}
```

`? super Integer`는 `Integer` 또는 `Integer`의 부모 타입 리스트를 받겠다는 뜻이다.

```java
List<Integer> ints = new ArrayList<>();
List<Number> numbers = new ArrayList<>();
List<Object> objects = new ArrayList<>();

addIntegers(ints);
addIntegers(numbers);
addIntegers(objects);
```

`super`는 쓰기 중심이라고 기억하면 좋다. `Integer`는 `Integer`, `Number`, `Object` 자리에 안전하게 들어갈 수 있기 때문이다.

## 11. extends와 super 정리

| 문법 | 의미 | 주 사용 목적 |
|---|---|---|
| `List<?>` | 어떤 타입인지 모르는 리스트 | 여러 타입 리스트 출력 |
| `List<? extends Number>` | Number 계열 리스트 | 읽기 중심 |
| `List<? super Integer>` | Integer의 부모 계열 리스트 | 쓰기 중심 |

짧게 기억하려면 다음처럼 정리할 수 있다.

- `extends`: 꺼내서 읽기 좋다.
- `super`: 넣어서 쓰기 좋다.

## 12. 전체 예제

```java
import java.util.ArrayList;
import java.util.List;

public class GenericWildcardMain {
    public static void printAll(List<?> list) {
        for (Object item : list) {
            System.out.println(item);
        }
    }

    public static void printNumbers(List<? extends Number> list) {
        for (Number item : list) {
            System.out.println(item);
        }
    }

    public static void addIntegers(List<? super Integer> list) {
        list.add(10);
        list.add(20);
    }

    public static void main(String[] args) {
        List<String> names = List.of("kim", "lee");
        printAll(names);

        List<Integer> numbers = List.of(1, 2, 3);
        printNumbers(numbers);

        List<Number> result = new ArrayList<>();
        addIntegers(result);
        System.out.println(result);
    }
}
```

## 연습 문제

1. `Box<T>` 클래스를 직접 작성해 보자.
2. `Box<String>`과 `Box<Integer>`를 각각 만들어 보자.
3. `NumberBox<T extends Number>`를 만들고 `Integer`, `Double`을 넣어 보자.
4. 배열과 값을 받아 같은 값의 개수를 반환하는 제네릭 매서드를 만들어 보자.
5. `List<?>`를 매개변수로 받아 모든 값을 출력하는 매서드를 만들어 보자.
6. `List<? extends Number>`와 `List<? super Integer>`의 차이를 직접 설명해 보자.
