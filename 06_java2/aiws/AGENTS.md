# AIWS Agent Guide

## Purpose

이 폴더는 Java2 학습을 돕기 위한 Markdown 자료와 예제 설명을 만드는 공간이다. 앞으로 이 폴더에서 작업하는 에이전트는 자바 초급 학습자가 이해할 수 있도록 설명을 작성한다.

## Learner Profile

- 사용자는 MVC2 패턴을 활용한 웹 개발을 학습한 뒤 Java2를 배우는 중이다.
- 인터페이스, 정렬, 제네릭, 와일드카드 개념을 실습 중심으로 이해하려고 한다.
- 설명은 한국어로 작성하고, 어려운 용어는 쉬운 말로 먼저 풀어서 설명한다.

## Writing Rules

- Markdown 문서는 한국어로 작성한다.
- 한 문단에는 하나의 핵심 생각만 담는다.
- 개념 설명 뒤에는 반드시 짧은 Java 예제를 붙인다.
- 예제는 초급자가 복사해서 실행해 볼 수 있을 정도로 단순하게 유지한다.
- 기존 수업 표현을 존중한다. 예: `매서드`, `인터페이스`, `제네릭`, `와일드카드`, `ArrayList<Acorn>`.
- 코드가 길어질 때는 먼저 핵심 코드만 보여 주고, 이후 전체 예제를 제공한다.
- 표는 비교가 필요한 경우에만 사용한다.

## Java Example Rules

- Java 예제는 가능하면 하나의 파일에서 이해할 수 있게 작성한다.
- import 문이 필요한 예제는 import 문까지 포함한다.
- `Comparable` 예제는 객체 자신의 기본 정렬 기준으로 설명한다.
- `Comparator` 예제는 외부에서 정렬 기준을 제공하는 방식으로 설명한다.
- 제네릭 예제는 `Box<T>`, `NumberBox<T extends Number>`, 제네릭 매서드, 와일드카드 순서로 설명한다.
- 와일드카드는 `List<?>`, `List<? extends T>`, `List<? super T>` 순서로 설명한다.

## Preferred Structure For Study Notes

각 학습 문서는 가능하면 다음 순서를 따른다.

1. 학습 목표
2. 왜 필요한가
3. 핵심 문법
4. 짧은 예제
5. 전체 예제
6. 정리
7. 연습 문제

## Scope Discipline

- 요청받은 Java2 학습 범위를 벗어난 고급 주제는 길게 확장하지 않는다.
- 스트림, 람다, 디자인 패턴은 필요한 경우에만 짧게 연결해서 설명한다.
- 초급 단계에서 혼란을 줄 수 있는 과도한 예외 처리, 복잡한 패키지 구조, 빌드 도구 설명은 피한다.

## Verification

Markdown 파일을 작성하거나 수정한 뒤에는 다음을 확인한다.

- 파일이 `06_java2/aiws` 안에 생성되었는지 확인한다.
- 코드 블록의 백틱이 닫혔는지 확인한다.
- 제목과 학습 목표가 요청한 내용과 맞는지 확인한다.
- 설명이 너무 추상적이면 예제를 추가한다.
