static 매서드 객체 매서드

객체 매서드는 객체에서 사용되므로 static을 제거한다 
public static void가 아닌 public void로


//****이부분 다시정리
만일 객체 class에서 static을 사용하게 되면 해당 매서드는 객체마다 존재하는 매서드가 아닌 전체 class에서 단 한개만 존재하는 매서드가 된다. 

=> 복수개의 객체를 만들어도 해당 매서드는 class에서만 발생하기 때문에 나중에 선언된 한개로의 값만 가진다. 

*****//


static 매서드는 객체 없이 호출가능, this를 사용할 수 없음. this는 객체를 의미하기 때문

따라서, 매서드를 static으로 쓸지 instance로 쓸지는 객체마다 변하는 값인지, 전체 class에서 하나로 작용하는 값인지를 
확인 한 후에 판단하면 된다고 이해함


만일 static 변수를 쓰게 되면....... 해당 변수에 단일로 값을 넣을수가 있음......... 
// static 변수는 static 매서드로 담으면 된다. 

AcornStudent장해든_static s = new AcornStudent장해든_static();
	
	s.입력하기("장해든", 1000, 30, 30, 100);
	
	s.정보출력하기();
	s.놀기놀기(); // => 객체의
	AcornStudent장해든_static.행복지수=50000; 
	s.정보출력하기();
	 

이런 경우에는 "클래스명.변수명"으로 호출해야함!
