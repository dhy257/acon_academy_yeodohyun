========================== 캡슐화 ==========================

✅  객체 : (구조화된 데이터 + 함수) 형태
		new를 통해 클래스에 적힌 각 속성과 함수를 탑재한 기계가 heap 영역에 생성

✅ 캡슐화 : 하나의 클래스 아래에서 관련이 있는 데이터와 그 데이터를 사용하는 함수(매서드)를 하나의 묶음으로 사용하는 것

캡슐화를 사용하는 이유: 보안 및 데이터 검증(변수에 한 순간이라도 이상한 데이터 들어가지 못하게) 
				  캡슐화로 만들어진 객체들은 퍼즐처럼 짜맞추기 편리하다는 장점이 있음
				  각 객체의 역할이 달라 각 함수와 속성을 통해 역할 분담 및 처리

=> 보안) 데이터를 외부에 직접 공개하지 않고 함수(매서드)를 통해서만 데이터를 사용하도록 하는 것

캡슐화 방식

- 틀 만들기
 
String name; 

- 매서드 만들기 
   	  구조화된 변수를 다루는 함수 사용
	  this키워드 제공 

         ✅ this:  객체 자신의 변수(속성)을 가르킬 수 있음
                      this를 사용하는 이유: 참조변수가 내 속성(변수)와 이름이 같을 경우 서로를 구분하기 위함

public void 입력하기(String name) {
		
		this.name=name;
	}


- 참조형 변수를 만들어서 객체를 사용한다

Snack2 s=new Snack2(); 





public class Snack {    
	
	
	어떻게 변수가 만들어질지 설계
	String name; 
	
	
	이 변수들을 어떻게 사용할지 설계
	
	public void 입력하기(String name) {
		
		this.name=name;
	}
	
	public void 출력하기() {
		System.out.println(this.name);
		
	}

}

public static void main(String[] args) {
		
		Snack2 s=new Snack2(); 
		
		s.입력하기("맛동산", "해태", "500", 200); 
		s.출력하기(); 		

	}


========================== 객체배열 ==========================

✅ 객체 배열: 다수개의 객체는 객체배열로 다뤄야 한다

Snack [] arr = new Snack [3];

객체를 참조할 수 있는 참조형변수를 배열로 만든 것이며, 객체가 만들어진 것이 아님
참조형변수의 기본값인 null로 초기화됨

========================== 생성자 ==========================

데이터보호를 위해서(무분별한 입력을 막기위해) 2가지 도구 제공

✅  접근제한자: private, public, default (같은폴더 생략)

		public:같은 프로젝트 안이라면 어떤 패키지든 접근 가능
		private:오직 자기가 선언된 클래스 내부에서만 사용 가능
		default(키워드 생략):같은 패키지 안이라면 접근 가능

✅  생성자 : new에 의해서 생성될 떄 호출되는 특별한 매서드
	      변수(객체)의 값을 초기화
              별도로 호출불가
		
		- 모든 클래스는 기본생성자를 제공 (기본생성자 = 매개변수가 없는 생성자)
		★ 생성자의 이름: 반드시 클래스 이름과 동일
		★ 반환 타입을 명시 x
   		-  오버로딩: 매개변수가 다른 생성자 여러 개 정의
		생성자를 만들면 기본생성자는 제공되지 않는다

생성자를 사용하는 이유 

- 객체 생성과 동시에 초기화
- 불필요한 객체내용 변경 차단
- 필수값을 반드시 받도록 강제 가능 (안정성 증가)

   ex)	기본생성자: public AcornStudent2() { //매개변수 없는 생성자
  				}
  
        생성자: public AcornStudent2(String name, int 에너지, int 근육량, int 자바능력, int 프론트능력){
		  		this.name  = name;
				this.에너지 = 에너지;
				this.근육량 = 근육량;
				this.자바능력 =  자바능력;
				this.프론트능력  = 프론트능력;
				  }
  

========================== 변수의 종류 ==========================

✅ 지역변수	: 매서드 안에서 만들어지는 변수
		  매서드 호출시 만들어지고 매서드 반환시에 사라짐 (메모리 반납)
		  매서드  내부에 작성
		
✅ static 변수 (class 변수) : 프로그램 시작 ~ 종료
				클래스 내부에 작성
				클래스내에서 공유할 목적의 값을 유지할 때
 				객체가 공유해야하는 값이 있을 때
				객체생성과 무관하게 사용할 수 있음
 static 변수 초기화 
		   1.기본값으로 초기화 
		   2.명시적 초기화
		   3.스태틱 초기화 블럭 : 복잡한 초기화 
		     static {
		     
		     }

		
✅ instance 변수(객체의 각 속성) : new에 의해서 만들어지고 참조를 잃으면 가비지의 대상이 됨
					   클래스 내부에 작성 
					   
instance 변수 초기화
		1. 아무것도 명시하지 않으면 자료형 기본값
		 	int => 0 double => 0.0
		2. 명시적초기화
			 int kor = 100;
		3. 초기화블럭(인스턴스 초기화 블럭){ } : 
		4. 생성자()
