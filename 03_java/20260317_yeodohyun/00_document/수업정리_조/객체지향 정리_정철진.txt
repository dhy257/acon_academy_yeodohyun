1. 생성자란?

객체 생성 시 실행되는 메서드
클래스 이름과 동일한 이름을 가짐
반환 타입이 없음

class Person {
    String name;

    Person(String name) {  // 생성자
        this.name = name;
    }
}

2. 특징

new 키워드로 객체 생성 시 자동 호출
반환값 없음
초기값 설정에 주로 사용

3. 기본 생성자
생성자를 하나도 안 만들면 컴파일러가 자동 생성

=> this ?
1. 기본 개념
객체 내부에서 자기 자신을 가리킬 때 사용
class Person {
    String name;

    Person(String name) {
        this.name = name; // this.name → 객체의 변수
    }
}
=> 이름이 같을때 구분해주기 위한 역할이라고 봐도 되기는함

단 static 메서드에서는 사용 불가
객체가 없이 실행되는 static에서 this로 현재 객체를 찾을 수는 없음

static void test() {
    Person p = new Person("홍길동");
    System.out.println(p.name); // ✅ 가능
}

가능하게 하려면 객체를 직접 만들어서 사용해야함

1. 캡슐화란?

간단히 말하면 데이터(변수)와 기능(메서드)을 하나로 묶고, 외부에서 직접 접근을 제한하는 것
“내부는 숨기고, 필요한 것만 보여준다”


class Person {
    String name;
}

Person p = new Person();
p.name = null; // 아무 값이나 막 넣음
이러면 아무 값이나 막 넣을 수 있음
=> 데이터 무결성이 깨짐


class Person {
    private String name;

    public void setName(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }
}

이렇게 하면 외부에서 직접 접근하지 못하고 메서드를 통해서만 접근 가능

보통은
public void setName(String name) {
    if (name == null) return; // 방어 로직
    this.name = name;
}
이렇게 아무값이나 들어오는걸 방지할때도 많이 사용

접근 제한자 (Access Modifier)
접근 제한자는 클래스, 변수, 메서드의 노출 범위를 결정.

private  ➡️  default  ➡️  protected  ➡️  public

private=>해당 클래스 내부에서만 접근 가능
default=>아무것도 안 적었을 때의 기본값, 같은 패키지 안에서만 접근 가능
protected=>같은 패키지 + 다른 패키지에 있더라도 상속받은 자식 클래스면 접근 가능
public=>어디서든 접근 가능
